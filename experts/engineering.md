## 주요 프레임워크 & 개념

### Will Larson의 스태프 엔지니어 아키타입
1. **Tech Lead**: 팀의 기술 방향 설정
2. **Architect**: 시스템 아키텍처 설계
3. **Solver**: 복잡한 문제 해결
4. **Right Hand**: 임원의 기술 파트너

### Camille Fournier의 관리자 경로
- **Tech Lead**: 기술 + 약간의 관리
- **Engineering Manager**: 팀 관리 (5-10명)
- **Director**: 여러 팀 관리
- **VP of Engineering**: 조직 전체
- **CTO**: 기술 전략 + 조직

### Nicole Forsgren의 DORA 메트릭스
4가지 핵심 지표:
1. **배포 빈도 (Deployment Frequency)**: 얼마나 자주 배포하는가
2. **리드 타임 (Lead Time)**: 코드 커밋부터 프로덕션까지 시간
3. **MTTR (Mean Time to Restore)**: 장애 복구 시간
4. **변경 실패율 (Change Failure Rate)**: 배포 중 실패 비율

### Gergely Orosz의 실용적 엔지니어
특징:
- 비즈니스 이해
- 제품 마인드
- 효과적인 커뮤니케이션
- 지속적 학습

## 엔지니어링 커리어 경로

### 개인 기여자 (IC) 트랙
```
Junior → Mid-level → Senior → Staff → Principal → Distinguished
```

### 관리 트랙
```
Tech Lead → EM → Senior EM → Director → VP → CTO
```

### 전환 포인트
- Senior에서 Staff vs EM 선택
- 언제든 트랙 전환 가능 (하지만 쉽지 않음)
- 자신의 강점과 관심사 고려

## 기술 부채 관리

### 기술 부채의 종류
1. **의도적 부채**: 빠른 출시를 위한 선택
2. **비의도적 부채**: 지식 부족으로 인한 부채
3. **환경 부채**: 외부 변화로 인한 부채

### 관리 전략
- 정기적인 리팩토링 시간 확보
- 기술 부채 가시화
- 비즈니스 임팩트 기준 우선순위
