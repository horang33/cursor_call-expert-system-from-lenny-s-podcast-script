## 주요 프레임워크 & 개념

### Chip Huyen의 ML 시스템 설계
ML 시스템 구성 요소:
1. **데이터 파이프라인**: 수집, 정제, 변환
2. **모델 학습**: 실험, 학습, 평가
3. **모델 배포**: 서빙, 모니터링, 업데이트
4. **피드백 루프**: 사용자 데이터로 지속 개선

### LLM 제품 개발 패턴
1. **Wrapper**: LLM API를 감싸서 특정 용도로 제공
2. **Agent**: LLM이 도구를 사용하여 작업 수행
3. **Copilot**: 사용자와 협업하는 AI 어시스턴트
4. **Autonomous**: 완전 자동화된 AI 시스템

### AI 제품 평가 프레임워크
- **정확성 (Accuracy)**: 올바른 결과를 제공하는가?
- **신뢰성 (Reliability)**: 일관된 성능을 보이는가?
- **지연시간 (Latency)**: 응답 속도가 적절한가?
- **비용 (Cost)**: 운영 비용이 합리적인가?

## AI 제품 개발 단계

### 1단계: 문제 정의
- AI가 정말 필요한가?
- 전통적 방법으로 해결 불가능한가?
- 성공 지표는 무엇인가?

### 2단계: 데이터 확보
- 필요한 데이터 식별
- 데이터 수집 전략
- 데이터 품질 검증

### 3단계: 프로토타입
- 간단한 베이스라인 모델
- 빠른 실험과 반복
- 사용자 피드백 수집

### 4단계: 프로덕션
- 확장 가능한 인프라
- 모니터링 시스템
- A/B 테스팅

### 5단계: 지속 개선
- 성능 모니터링
- 재학습 파이프라인
- 새로운 데이터 통합

## AI 시대의 제품 전략

### 차별화 요소
1. **데이터**: 고유한 데이터 자산

3. **사용자 경험**: AI를 자연스럽게 통합
