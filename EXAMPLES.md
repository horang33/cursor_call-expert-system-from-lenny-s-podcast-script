## Product Expert - Product Management

### 예시 1: 제품-시장 적합성 찾기

**질문:**
```
/lenny 프로덕트 마켓 핏을 어떻게 찾나요? 초기 스타트업입니다.
```

**기대 응답:**
```
[자동 선택: Product Expert]

Brian Chesky, Rahul Vohra, Sean Ellis의 관점에서 답변드리겠습니다.

프로덕트 마켓 핏(PMF)은 "사용자가 제품 없이는 살 수 없다"고 느끼는 상태입니다.

구체적으로:

1. **40% 테스트 (Sean Ellis)**
   - 사용자에게 질문: "이 제품이 없어진다면 어떻게 느끼시겠습니까?"
   - 40% 이상이 "매우 실망할 것"이라고 답하면 PMF 달성
   - 이 지표는 Dropbox, Superhuman 등에서 검증됨

2. **리텐션 커브 (Casey Winters)**
   - 코호트 리텐션 커브가 평평해지는 지점 찾기
   - 사용자들이 계속 돌아오는가?
   - 주간 활성 사용자 추적

3. **유기적 성장 (Brian Chesky)**
   - 사용자들이 자발적으로 추천하기 시작
   - 입소문이 자연스럽게 발생
   - 마케팅 없이도 성장

Brian Chesky가 "Airbnb의 초기 성장" 에피소드에서 강조한 것처럼, PMF 이전에는 
확장(scaling)보다 제품 개선에 집중해야 합니다. 소수의 열성 사용자를 만드는 것이 
수천 명의 미온적 사용자보다 중요합니다.

참고 에피소드:
- Brian Chesky - Airbnb의 성장 스토리
- Rahul Vohra - Superhuman의 PMF 여정
- Sean Ellis - 40% 테스트 방법론
```

### 예시 2: 제품 우선순위 결정

**질문:**
```
/lenny 제품 기능 우선순위를 어떻게 결정해야 하나요? 
요청이 너무 많아서 혼란스럽습니다.
```

**기대 응답:**
```
Shreyas Doshi와 Gibson Biddle의 관점에서 답변드리겠습니다.

우선순위 결정은 PM의 가장 중요한 스킬입니다.

**Shreyas Doshi의 LNO 프레임워크:**

1. **Leverage (레버리지)**: 큰 임팩트를 만드는 작업
   - 20%의 시간으로 80%의 결과
   - 예: 핵심 사용자 문제 해결, 차별화 기능
   - 이것에 최대한 많은 시간 투자

2. **Neutral (중립)**: 필요하지만 차별화되지 않는 작업
   - 예: 테이블 스테이크 기능, 버그 수정
   - 빠르게 처리하거나 외주

3. **Overhead (오버헤드)**: 꼭 필요한 관리 작업
   - 예: 보고, 미팅, 프로세스
   - 최소화하고 자동화

**Gibson Biddle의 DHM 모델:**
각 기능을 평가:
- Delight: 고객을 기쁘게 하는가?
- Hard-to-copy: 경쟁사가 복제하기 어려운가?
- Margin-enhancing: 비즈니스 수익성을 높이는가?

세 가지 모두 "예"인 기능을 우선순위로.

**실전 팁:**
- 모든 요청을 LNO로 분류하세요
- Leverage 작업에 80% 시간 투자
- "No"라고 말하는 연습하기
- 우선순위는 전략에서 나옴

참고 에피소드:
- Shreyas Doshi - 제품 우선순위 결정
- Gibson Biddle - Netflix 제품 전략
```

## Strategy Expert - Product Strategy

### 예시 1: OKR 설정

**질문:**
```
/lenny 팀 OKR을 처음 설정하려고 합니다. 어떻게 시작해야 하나요?
```

**기대 응답:**
```
Christina Wodtke와 Shishir Mehrotra의 관점에서 답변드리겠습니다.

OKR(Objectives and Key Results)은 목표 설정과 실행을 연결하는 프레임워크입니다.

**OKR 구조:**

**Objective (목표)**:
- 영감을 주는 정성적 목표
- 팀이 달성하고 싶은 것
- 예: "모바일 사용자 경험을 혁신한다"

**Key Results (핵심 결과)**: 3-5개
- 측정 가능한 정량적 결과
- 목표 달성을 나타내는 지표
- 예: 
  1. 모바일 앱 NPS를 40에서 60으로 향상
  2. 모바일 전환율을 2%에서 4%로 증가
  3. 모바일 로딩 시간을 3초에서 1초로 단축

**Christina Wodtke의 조언:**

1. **주간 체크인**: 매주 진행상황 확인
2. **4가지 질문**:
   - 지난주 무엇을 했나?
   - 이번주 무엇을 할 것인가?
   - 무엇이 막고 있나?
   - 배운 것은 무엇인가?

3. **흔한 실수**:
   - 너무 많은 OKR (3-5개로 제한)
   - 측정 불가능한 KR
   - 일상 업무를 OKR로 착각
   - 체크인 안 함

**Shishir Mehrotra의 Functional vs Ceremonial:**
- Functional OKR: 실제 의사결정에 사용
- Ceremonial OKR: 형식적인 문서
- Functional OKR을 만들어야 함!

**실전 팁:**
1. 회사 OKR → 팀 OKR → 개인 OKR 순서로 정렬
2. 주간 체크인 필수
3. 분기말 회고
4. 70-80% 달성이 목표 (100%는 너무 쉬운 것)

참고 에피소드:
- Christina Wodtke - OKR 실행 방법
- Shishir Mehrotra - 효과적인 목표 설정
```

## Leadership Expert - Leadership

### 예시 1: 1:1 미팅

**질문:**
```
/lenny 효과적인 1:1 미팅은 어떻게 진행하나요? 
신입 관리자입니다.
```

**기대 응답:**
```
Matt Mochary와 Kim Scott의 관점에서 답변드리겠습니다.

1:1 미팅은 관리자의 가장 중요한 도구입니다.

**Matt Mochary의 1:1 프레임워크:**

1. **Check-in (5분)**: 개인적 안부
   - "어떻게 지내세요?"
   - 업무 외 삶도 관심 갖기
   - 신뢰 구축

2. **Priorities (10분)**: 이번 주 우선순위
   - "이번 주 가장 중요한 3가지는?"
   - 정렬 확인
   - 방향 수정

3. **Progress (10분)**: 지난 주 진행상황
   - 완료한 것
   - 배운 것
   - 축하하기

4. **Problems (15분)**: 막힌 부분
   - "무엇이 당신을 막고 있나요?"
   - "제가 어떻게 도울 수 있나요?"
   - 장애물 제거

5. **Personal (10분)**: 커리어, 성장
   - 장기 목표
   - 스킬 개발
   - 피드백

**Kim Scott의 Radical Candor:**
- Care Personally: 개인적으로 관심 갖기
- Challenge Directly: 직접적으로 도전하기
- 두 가지 모두 필요!

**실전 팁:**
- 주간 30-60분 일정 고정
- 팀원이 아젠다 주도하도록
- 메모 작성 (공유 문서)
- 절대 취소하지 않기
- 경청이 80%, 말하기 20%

**피해야 할 것:**
- 상태 업데이트만 (이메일로 가능)
- 관리자가 주도
- 메모 없음
- 자주 취소

Matt Mochary가 Coinbase, Notion 등의 CEO들을 코칭하면서 강조한 것처럼,
일관된 1:1은 신뢰 구축과 성과 향상의 핵심입니다.

참고 에피소드:
- Matt Mochary - CEO 코칭 방법
- Kim Scott - Radical Candor
```

## Growth Expert - Growth Strategy

### 예시 1: 리텐션 개선

**질문:**
```
/lenny 리텐션이 낮습니다. 어떻게 개선해야 하나요?
```

**기대 응답:**
```
Casey Winters와 Brian Balfour의 관점에서 답변드리겠습니다.

리텐션은 성장의 기반입니다. 리텐션 없이는 지속 가능한 성장이 불가능합니다.

**1. 리텐션 측정:**

**Casey Winters의 L7/L30:**
- L7: 지난 7일 중 며칠 사용했는가
- L30: 지난 30일 중 며칠 사용했는가
- 제품 유형에 따라 다름:
  - 소셜: 매일 (L7 = 5-7)
  - 유틸리티: 주 2-3회
  - 마켓플레이스: 월 1-2회

**리텐션 커브:**
- Day 1, Day 7, Day 30, Day 90 리텐션 추적
- 커브가 평평해지는 지점 = PMF 신호
- 코호트별로 분석

**2. 리텐션 개선 전략:**

**Brian Balfour의 접근법:**

a) **Aha Moment 최적화**
   - 사용자가 가치를 처음 경험하는 순간
   - 예: Facebook - 10일 내 7명 친구 추가
   - 이 순간까지 시간 단축

b) **습관 형성**
   - 트리거 → 행동 → 보상 루프
   - 알림, 이메일로 트리거
   - 즉각적인 가치 제공

c) **기능 채택**
   - 핵심 기능 사용률 높이기
   - 온보딩 개선
   - 인앱 가이드

**3. 코호트 분석:**

주간 코호트로 분석:
- 어떤 코호트가 리텐션 높은가?
- 무엇이 달랐는가?
- 재현 가능한가?

**4. 실행 우선순위:**

1. Aha Moment까지 시간 단축 (가장 중요)
2. 핵심 기능 채택률 높이기
3. 재참여(Re-engagement) 캠페인
4. 제품 개선

Casey Winters가 Pinterest에서 강조한 것처럼, 
"리텐션이 성장의 엔진입니다. 획득에 투자하기 전에 리텐션을 고치세요."

참고 에피소드:
- Casey Winters - 마켓플레이스 성장
- Brian Balfour - 성장 루프
```

## 💡 효과적인 질문 작성 팁

### 1. 구체적으로 질문하기

❌ **나쁜 예:**
```
/lenny 제품 관리가 뭔가요?
```

✅ **좋은 예:**
```
/lenny B2B SaaS 스타트업에서 첫 PM으로 입사했습니다. 
첫 90일 동안 무엇에 집중해야 하나요?
```

### 2. 상황 설명하기

✅ **좋은 예:**
```
/lenny 우리는 B2B SaaS 제품이고, 현재 MRR $50K입니다. 
다음 성장 단계로 가기 위해 어떤 채널에 집중해야 하나요?
```

### 3. 특정 게스트 언급

✅ **좋은 예:**
```
/lenny Matt Mochary의 1:1 프레임워크를 
우리 팀에 적용하고 싶습니다. 어떻게 시작해야 하나요?
```

### 4. 복합 질문

✅ **좋은 예:**
```
/lenny /lenny 
초기 스타트업에서 제품 개발과 성장을 어떻게 균형잡아야 하나요?
```
